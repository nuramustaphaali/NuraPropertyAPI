# Contributing to NuraPropertyAPI

First off, thank you for considering contributing to NuraPropertyAPI! It's people like you that make the open-source community such an amazing place to learn, inspire, and create.

## ü§ù How to Contribute

### 1. Fork the Repository
Click the "Fork" button at the top right of this page to create your own copy of the repository.

### 2. Clone your Fork
```bash
git clone [https://github.com/nuramustaphaali/NuraPropertyAPI.git](https://github.com/nuramustaphaali/NuraPropertyAPI.git)
cd NuraPropertyAPI
```

### 3. Create a Branch
Create a new branch for your feature or bugfix. Please use descriptive names.
```bash
git checkout -b feature/add-payment-gateway
# or
git checkout -b fix/login-error
```

### 4. Make your Changes
Write your code! Please follow these guidelines:
* Follow PEP 8 style guidelines for Python code.
* Ensure you create a new App if the feature is distinct (Modular Architecture).
* Add comments where necessary.

### 5. Test your Changes
Run the server and test your endpoints using Swagger (`/api/docs/`) or Postman.

### 6. Commit and Push
```bash
git add .
git commit -m "feat: Add Paystack integration for payments"
git push origin feature/add-payment-gateway
```

### 7. Submit a Pull Request (PR)
Go to the original repository and click "Compare & pull request". Provide a clear description of what you changed.
